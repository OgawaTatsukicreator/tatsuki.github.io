# 月間シフト提出サイト

1か月分のシフトをまとめて提出し、Googleスプレッドシートに保存する静的HTMLサイトです。
提出期限は毎月15日として画面と送信データに入っています。

## ファイル

- `index.html`: 月間シフト提出画面
- `styles.css`: 画面デザイン
- `app.js`: 月間カレンダー生成、下書き保存、送信処理
- `Code.gs`: Google Apps Scriptに貼り付けるバックエンド

## Googleスプレッドシート連携手順

1. 対象スプレッドシートを開きます。
2. メニューの「拡張機能」から「Apps Script」を開きます。
3. `Code.gs` の内容を貼り付けて保存します。
4. Apps ScriptでHTMLファイルを追加し、ファイル名を `GasApp` にします。
5. `GasApp.html` の内容を貼り付けて保存します。
6. 「デプロイ」から「新しいデプロイ」を選びます。
7. 種類は「ウェブアプリ」にします。
8. 実行ユーザーは「自分」、アクセスできるユーザーは提出者に合わせて設定します。
   - スマホから誰でも提出できるようにするなら「全員」にします。
   - Googleログインが必要でよければ、組織やアカウント範囲に制限してください。
9. デプロイ後に発行されたウェブアプリURLをスマホで開きます。

GAS版は `google.script.run` で送信するため、`app.js` の `SCRIPT_URL` 設定は不要です。

## スプレッドシートに作成されるシート

- `シフト提出`: 送信された個人別の元データ
- `全体_YYYY-MM`: 月ごとの全店舗集計
- `A店_YYYY-MM` など: 月ごとの店舗別集計

同じ「名前・店舗名・対象月」で再申請すると、`シフト提出` の同じ行を更新し、月別集計シートも作り直します。

## 変更できる仕組み

名前、連絡先、対象月が同じ提出は、スプレッドシート上の同じ行を更新します。
そのため、スタッフは同じ情報で再申請すればいつでも変更できます。

## 入力区分

- `勤務`: 開始・終了時間を保存します。初期値は 08:00-22:00 です。
- `公休`: 休みとして保存します。
- `PT`: パーソナルを行う日として保存します。
- `相談`: 時間や条件を相談したい日として保存します。
- `未申請`: 未入力扱いです。

## 受付停止

締切後に受付を止めたい場合は、Apps Script側の `ACCEPTING` を `false` に変更して再デプロイしてください。
